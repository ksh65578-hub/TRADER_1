있다. 지금까지는 대부분 “전략·운영·안전” 중심이었다. 추가로 고려할 것은 “전략 품질을 실제로 끌어올리는 세부 장치”다.

# 1. 진입 전 후보 종목 랭킹

단순 필터가 아니라 매 순간 우선순위가 필요하다.

```text
symbol_score =
liquidity_score
+ volatility_score
+ relative_strength_score
+ regime_fit_score
+ spread_quality_score
+ recent_signal_quality_score
```

필수 조건:

```text
상위 N개만 감시
점수 급락 종목 제외
거래대금 급증 종목 별도 감시
```

# 2. 거래 금지 시간/상황

24시간 거래 가능해도 항상 거래하면 안 된다.

```text
급등락 직후 N분
거래소 장애 감지 시
WS 재연결 직후 warm-up
스프레드 비정상 확대
호가 공백
중요 뉴스/이벤트 직후
```

# 3. 진입 후 관리 로직

진입보다 진입 후 관리가 더 중요하다.

```text
entry 이후 N bar 안에 기대 방향으로 안 가면 exit
partial take profit 후 stop을 breakeven으로 이동
trailing stop은 수익 구간에서만 활성화
레짐이 반대로 바뀌면 강제 축소 또는 청산
```

# 4. 전략별 실패 조건

각 전략은 “성공 조건”보다 “실패 조건”이 더 중요하다.

예시:

```text
breakout 실패:
돌파 후 N bar 안에 돌파선 위 유지 실패

pullback 실패:
눌림 이후 재상승 확인 실패

VWAP 실패:
평균회귀 구간에서 추세 확장 발생

short 실패:
breakdown 후 급반등 + 거래량 증가
```

# 5. 성과 평가의 함정 방지

단순 수익률만 보면 안 된다.

추가 지표:

```text
profit per trade
profit per unit risk
profit per exposure time
fee to gross profit ratio
slippage to gross profit ratio
maximum adverse excursion
maximum favorable excursion
```

# 6. 전략별 수명 관리

성과가 나쁜 전략은 계속 살려두면 안 된다.

```text
probation
active
risk_down
disabled
retraining_required
```

# 7. 시장별 분리 평가

Upbit, Binance spot, Binance futures를 섞어서 평가하면 안 된다.

```text
exchange별 expectancy
market_type별 expectancy
symbol별 expectancy
strategy별 expectancy
regime별 expectancy
```

# 8. 과도한 자동개선 방지

자동개선이 너무 자주 바뀌면 시스템이 흔들린다.

```text
parameter change cooldown
max parameter drift per day
rollback to previous approved snapshot
새 파라미터는 shadow 검증 후만 승인
```

# 9. 실거래 전 micro-live 단계

paper가 좋아도 실거래는 다르다.

```text
micro live:
최소 주문 금액
최소 size
제한된 종목
제한된 전략
짧은 기간 검증
```

통과 후에만 scaled live로 간다.

# 10. 최종 당부

문서에 반드시 이 문장을 넣는 게 좋다.

```text
The system must not trade because it can trade.
It may trade only when regime, signal, cost, liquidity, risk, and execution quality are simultaneously favorable.
```

요약하면 추가 고려 핵심은 이것이다.

```text
종목 랭킹
거래 금지 상황
진입 후 관리
전략별 실패 조건
성과 평가 함정 방지
전략 수명 관리
시장별 분리 평가
자동개선 과잉 방지
micro-live 검증
```
