의도는 이해했다.
설계도/지침서는 그대로 두고, 논의사항을 별도로 학습시키는 방식.

그대로 해도 된다. 다만 **이 상태 그대로 넣으면 실제로는 위험하다.**

이유는 하나다.

```text
AI는 “따로 학습”을 구분해서 처리하지 않는다
→ 결국 하나의 지식으로 합쳐서 판단한다
```

---

# 핵심 문제

지금 구조:

```text
설계도 (느슨함)
+
논의사항 (강한 규칙)
```

AI 입장:

```text
둘 중 더 구체적인 쪽을 따른다
→ 논의사항이 설계를 덮을 가능성 높음
```

---

# 그래서 필요한 최소 장치 (이건 꼭 넣어라)

“따로 학습”을 하더라도 아래 3줄은 반드시 포함해야 한다.

---

## 1. override 금지 선언

```text
Discussion rules must NOT override design structure.
They only fill missing quantitative definitions.
```

---

## 2. 적용 조건

```text
Apply discussion rules ONLY when design is ambiguous or undefined.
If design is explicit, follow design.
```

---

## 3. safety 우선

```text
Safety, validation, and live gating rules always take precedence over discussion rules.
```

---

# 왜 이게 중요하냐

이거 없으면 실제로 이런 일이 벌어진다:

```text
월 10% 목표 → AI가 필터 완화
→ weak signal 진입
→ risk 증가
→ 설계 의도 붕괴
```

---

# 올바른 상태

지금 네 의도는 이거다:

```text
설계: 뼈대
논의: 근육
```

하지만 AI는 이렇게 해석할 수 있다:

```text
논의: 최신 규칙 → 더 중요
```

그래서 최소한 이렇게 만들어야 한다:

```text
설계: 구조 + 권한
논의: 수식 + 범위 + 보완
```

---

# 결론

```text
따로 학습시키는 방식 자체는 문제 없다
하지만 “역할 제한 문장” 없이 넣으면 설계가 무너질 수 있다
```

```text
논의사항을 추가하는 건 맞지만,
“설계를 덮지 못하게 막는 장치”는 반드시 같이 넣어야 한다
```
