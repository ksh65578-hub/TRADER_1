사용자의 목표는 단순한 “자동매매 프로그램”이 아니라, 다음과 같이 명확하게 정의됩니다.

---

# 1. 최종 목표 (핵심 정의)

사용자가 원하는 것은:

“사용자는 켜고 끄는 것만 하고
시스템이 알아서 수익을 최적화하는
장기 안정 자동매매 시스템”

---

# 2. 사용자 행동 모델 (매우 중요)

사용자가 실제로 하고 싶은 행동은 딱 이겁니다.

1. PAPER 실행
2. 대시보드 확인
3. LIVE_READY 확인
4. LIVE 실행
5. 필요 시 중지

끝입니다.

즉,

* 전략 생성 안 함
* 파라미터 튜닝 안 함
* 종목 선택 안 함
* 로그 분석 안 함

모든 판단은 시스템이 대신해야 합니다.

---

# 3. 시스템 역할 (사용자를 대신하는 부분)

## 3.1 시장 판단

* 상승장 / 횡보장 구분
* 하락장 회피 (핵심 요구사항)
* 적절한 전략 자동 선택

---

## 3.2 종목 선택

* 유동성 높은 종목 자동 선택
* 변동성 기반 필터링
* 저변동/비효율 종목 제외

---

## 3.3 전략 실행

* 눌림목 추세추종
* 돌파
* VWAP 기반 평균회귀
* 하이브리드 전략

---

## 3.4 자동 개선 (핵심)

* paper / shadow 실행 중 데이터 축적
* 파라미터 자동 조정
* 성과 비교
* 더 나은 전략으로 자연스럽게 이동

사용자는 절대 개입하지 않음

---

## 3.5 수익 최적화

* 리스크 제한
* 과도한 보수성 탈출 가능
* 데이터 기반으로 점점 공격적/보수적 조정

---

# 4. 거래 범위

시스템은 반드시 다음을 지원해야 합니다.

* Upbit (현물)
* Binance (현물)
* Binance (USDT-M 선물, 1x, 양방향)

즉:
“거래소 종속이 아닌 통합 트레이딩 시스템”

---

# 5. 운영 방식

## 필수 조건

* 24시간 장기 안정 실행
* 빠른 반응 속도
* 장애 발생 시 자동 복구
* 데이터 축적 지속

---

## 모드 구조

* PAPER
* SHADOW
* LIVE

흐름:

PAPER → SHADOW 병행 → 성과 검증 → LIVE_READY → LIVE

---

# 6. LIVE 진입 철학

중요한 철학:

* 무조건 수익 보장 X
* 데이터 기반으로 판단
* 충분한 검증 후 LIVE

즉,
“억지로 돈 벌게 만드는 시스템”이 아니라
“데이터로 검증된 전략만 live로 올리는 시스템”

---

# 7. 사용자 UX 요구

사용자가 보고 싶은 것:

* LIVE_READY 여부
* 현재 자산
* 포지션
* 진입 이유
* 진입 안 한 이유
* 시스템 정상 작동 여부

그리고:

“복잡한 건 숨기고 핵심만 보여야 함”

---

# 8. 설계 철학

가장 중요한 기준:

1. fail-closed (안전 최우선)
2. AI가 이어서 구현 가능해야 함
3. 완벽한 문서보다 “올바른 방향 유도”가 중요
4. 과도한 복잡도 금지
5. 현실 거래 제약 반드시 반영

---

# 9. 절대 하면 안 되는 것

* live에서 추측 기반 거래
* 검증되지 않은 전략 적용
* 사용자 수동 개입 요구
* 복잡한 UI
* 쓸데없는 고급 기능

---

# 10. 한 줄 요약

이 시스템은:

“사용자가 버튼만 누르면
시장이 어떤 상태든 알아서 대응하면서
점점 더 잘 벌도록 스스로 개선하는 자동매매 시스템”

---

# 핵심 진단 (현재 상태와 비교)

지금 시스템은:

* 안전 구조: 거의 완성
* 판단 구조: 없음
* 실행 구조: 없음
* 개선 구조: 없음

즉:

“생각하고, 거래하고, 개선하는 기능”이 없음
